# Claim Boundaries

This document separates what is implemented today from what is framing or future direction.

## Implemented Primary Demo

You can claim:

- TriageCore provides a local-first operator workflow.
- TriageCore generates reviewable preflight and handoff artifacts.
- TriageCore exposes privacy-safe route audit inspection.
- TriageCore includes a narrow Qwen Cloud adapter path for external-safe packets only.

Do not claim:

- that the judge quickstart proves live production cloud execution
- that every cloud-shaped route automatically calls Qwen

## Implemented Supporting Framing

You can claim:

- the project demonstrates a safer AI-assisted SDLC framing through explicit review, privacy boundaries, and auditability
- the repo contains implemented evidence for route auditing, handoff generation, and fail-closed local-only behavior

Do not claim:

- that the repo alone proves complete organizational governance
- that the documentation bundle replaces human review

## Future Extension

You can claim:

- the same local-first control pattern is a good fit for future environmental and edge workflows
- Clear Lake Watch is a plausible next extension area for this model

Do not claim:

- that Clear Lake Watch integration is already implemented in this repo
- that the current quickstart validates environmental field deployment outcomes

## Mocked-Test Boundary

You can claim:

- the Qwen adapter path is tested with mocked behavior
- the judge workflow does not require live Qwen credentials

Do not claim:

- that the quickstart demonstrates live remote service reliability

## Quick Summary

Use this if judges want the shortest version:

- implemented primary demo: local-first TriageCore with optional Qwen Cloud escalation
- implemented supporting framing: safer AI-assisted SDLC through bounded review and route auditing
- future extension: environmental edge workflows such as Clear Lake Watch
